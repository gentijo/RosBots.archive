TODO This implements depth-first left-to-right MRO, which is not compliant with Python3 MRO http://python-history.blogspot.com/2010/06/method-resolution-order.html https://www.python.org/download/releases/2.3/mro/  will keep lookup-#gt;dest[0]'s value (should be MP_OBJ_NULL on invocation) if attribute is not found will set lookup-#gt;dest[0] to MP_OBJ_SENTINEL if special method was found in a native type base via slot id (as specified by lookup-#gt;slot_offset). As there can be only one native base, it's known that it applies to instance-#gt;subobj[0]. In most cases, we also don't need to know which type it was - because instance-#gt;subobj[0] is of that type. The only exception is when object is not yet constructed, then we need to know base native type to construct its instance-#gt;subobj[0] from. But this case is handled via instance_count_native_bases(), which returns a native base which it saw.<br/>
